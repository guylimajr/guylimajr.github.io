Commands UI for OneNote

Overview

Microsoft OneNote is a notetaking application that supports a bunch of different ways to take notes.

We were completely overhauling our Windows 8 app for Windows 10 and needed a new way to present all of OneNote's commands.

With a colleague, I designed a new command interface that:

  • Conveyed what made OneNote special
  • Scaled to accommodate the commands being added for the Windows 10
  • Was easily learnable

I simplified the design task by shifting its focus from interaction design to information architecture.

This design shipped as part of the Windows 10 release in the Fall of 2015.

Design Goals

When I joined my colleague on this project, we defined three main goals for a new commands interface.

Show You What OneNote is Good For

When you start OneNote for the first time, it's hard to tell how it's different from other apps you already know and use, like Word.

The UI should show that OneNote helps you keep different types of notes.

Scale to Fit Lots of Commands

OneNote for Windows 10 was adding more features quickly as it moved to reach feature parity with OneNote 2013.

We needed a UI model that would be able to accommodate the many commands that existed and would soon be added.

Easily Learnable

The Windows 8 app used the Radial Menu for its commands. Usability studies showed many users struggled with the previous UI. Many never figured out how to fully use it.

We wanted new users to focus on exploring what the app was good for, not worrying about how to use it.

Background

My colleague had been working on this project for a little while before I joined. She was trying to solve problems specific to the previous Windows 8 app in addition to some that had existed across all OneNote clients for years.

Users Love That OneNote Keeps Many Kinds of Things

Microsoft OneNote is a notetaking application that lets you keep many different kinds of content, like

  • to do lists
  • scanned documents/whiteboards
  • audio recordings
  • file attachments
  • handwriting

OneNote's most avid users love that they can store whatever they want in OneNote.

But On First Use, It Looks A Lot Like Word

For years, user studies and interviews told us that people didn't really get what made OneNote different from Word. Starting OneNote for the first time, they'd notice:

  • A Ribbon filled with a lot of text editing commands, just like Word
  • A big white page where they can type, whereas Excel and PowerPoint have distinct canvases

They would just walk away from the app unsure of how it was any different than Word, which they already knew and used.

We wanted to clarify that OneNote was a great place to store todos, images, audio recordings, and files that you want to remember later.

Need To Replace The Radial Menu

OneNote for Windows 8 put its commands in a Radial Menu, which had two main problems:

  • Usability: many users never figured out that tapping the arrow in the outer sector of the Radial Menu would show more commands
  • Scalability: as OneNote grew, we were having issues adding commands; the Radial Menu could only house 8 per-level

We wanted to replace the Radial Menu with something that was easier to use and could accommodate the new features we were adding for the Windows 10 release.

Designing New Interactions Isn't Working

My colleague believed that elevating a few, key features to the top-level of the UI would help users' understand OneNote's unique value.

She had tested changing how you interact with the Ribbon, like making it hidden by default or adding an extra row of buttons.

However, things just weren't clicking in usability studies.

Focusing on Information Architecture

My colleague's previous efforts had focused on changing how you interacted with the Ribbon. When I joined the project, I explored if we could simply redesign its information architecture to better show OneNote's value.

People Have Learned The Ribbon

The Ribbon is a UI pattern that had been used throughout Office and Windows for over 7 years. Usability studies showed users were comfortable and capable using the standard Ribbon in OneNote apps due to their years of experience using it elsewhere. Changing how the Ribbon worked lead to usability issues as users had to learn something new.

Other Office apps showed that the Ribbon scaled to a huge number of commands.

As-is, the Ribbon solved solved usability and scalability. To simplify the design process, I proposed we clarify OneNote's value by changing the Ribbon's information architecture.

What's wrong with OneNote's Ribbon?

I formed the hypothesis that OneNote's Ribbon made the app feel like it was meant for creating polished documents, rather than for taking notes for yourself. I identified three main problems:

  • The default tab was filled with text formatting commands, which make documents look good but aren't needed for notetaking
  • That tab was called "Home", which made it feel like the place you're meant to spend much of your time.
  • Unique notetaking commands like insert picture, attach file, and record audio, were hidden in the "Insert" tab

Showing What Makes OneNote Special

I expiremented with different layouts that elevated OneNote's key features to the default view

  • Created a new default tab called "Notes"
  • Filled the "Notes" tab with notetaking-specific commands
  • Renamed the "Home" tab to "Formatting and deprioritized its order

Does It Work?

After sharing my work with my colleague, she created a low-fi prototype using PowerPoint and we ran user tests. We found:

  • Good: Users identified what made OneNote unique based on the commands presented at the top-level
  • Bad: Users expected to be able to do formatting from the "Home" tab, like they did in other apps

"Home" Remodel

Users had problems with formatting commands not being in the first tab, like they are in other Office apps. I set out to redesign the Home Tab to provide access to formatting commands, but emphasize the commands that made OneNote unique.

Prioritizing Commands Based On Data

I used a heatmap, created from analytics, to determine which commands were most-used in OneNote. This helped me ensure the right commands would be available at the top-level.

Merging Notes and Formatting

I sketched different information architectures that placed the top-used formatting commands and the uniquely notetaking commands at the top-level of the "Home" tab.

I placed the less-used formatting commands in popovers, at the second-level.

Making Sure It All Fits

My colleague and I created pixel-perfect mockups for how the UI would respond to different screen sizes.

This design added value across all platforms, so I created mockups for iOS. She handled Windows 10.

Results

This design shipped as part of OneNote's Windows 10 app in late 2015. Relying on the existing Ribbon paradigm let users rely on their existing knowledge and made it easy to add new commands as they were built.

User feedback showed that some users found the Radial Menu more delightful. However, usability studies showed that an overwhelming majority were more competent at using the updated Ribbon.

User tests showed that this new model greatly increased user identification of OneNote's main selling points.
